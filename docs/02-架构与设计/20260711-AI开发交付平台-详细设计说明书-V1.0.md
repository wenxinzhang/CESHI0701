## 二、引言

### 2.1 编写目的

本文档在 SRS 需求规格说明书基础上，给出 AI 开发交付平台的实现级设计，面向后端开发、前端开发与测试工程师，作为编码实现、数据库建表与接口联调的直接依据。文档内容与平台现有代码实现保持一致，覆盖模块类设计、数据库物理设计与 API 详细设计三个维度。

### 2.2 设计依据

- 《20260711-AI开发交付平台-SRS需求规格说明书-V1.0》
- 平台现有代码实现：后端 NestJS 11 + Prisma 6（MySQL），前端 Vue 3 + Element Plus
- Prisma 数据模型定义（`server/prisma/schema.prisma`，共 40 个模型）
- 后端接口实现（`server/src/modules/**/controllers`，共 27 个控制器）

### 2.3 全局约定

本节约定贯穿所有模块，各模块章节不再重复。**以下口径均以平台真实实现为准。**

**统一响应格式**

所有接口经统一响应拦截器包装为固定外壳：

```typescript
interface ApiResponse<T> {
  code: number      // 业务状态码：200 成功，400 业务错误，401 未登录，403 无权限，500 服务器错误
  message: string   // 提示信息，成功为 "success"
  data: T           // 业务数据，可为 null
}
```

分页接口的 `data` 内层结构为：

```typescript
interface PageData<T> {
  list: T[]                       // 当前页数据列表
  pagination: {
    page: number                  // 当前页码
    pageSize: number              // 每页条数
    total: number                 // 总记录数
  }
}
```

> 注：本平台成功状态码为 `200`（非 `0`），分页元信息包裹在 `data.pagination` 中，与部分通用模板的 `code:0`、平铺分页字段口径不同，实现与联调以本约定为准。

**公共字段口径**

平台数据表的公共字段实际为：

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INT AUTO_INCREMENT | 主键，自增 |
| createTime（列名 create_time） | DATETIME | 创建时间，插入时默认当前时间 |
| updateTime（列名 update_time） | DATETIME | 更新时间，更新时自动刷新 |
| tenantId（列名 tenant_id） | INT | 租户 ID，多数业务表含此字段用于租户隔离（部分单行配置表与关联表无） |

> 说明：平台未采用 `created_by / updated_by` 审计字段；操作者信息在需要处以业务字段（如 `creator`、`operator`、`userId`）单独承载。主键为自增 INT，非 UUID。

**命名规范**

- 表名：snake_case，带业务前缀（如 `base_sys_*`、`sys_agent_*`、`model_*`、`task_*`、`dict_*`、`space_*`）。
- 字段：Prisma 模型层 camelCase，数据库列名 snake_case（由 Prisma 映射）。
- 接口路径：管理端统一前缀 `/admin`，按模块分组（如 `/admin/ag-ui`、`/admin/security-policy`、`/admin/agent-memory`）；登录等开放接口在 `/admin/open`。
- 基础 CRUD 模块由统一基类提供 7 个标准动作：list / detail / add / update / update-status / delete / batch-delete。

**错误码规范**

统一以响应外壳的 `code` 表达：`200` 成功；`400` 业务错误（参数校验失败、唯一约束冲突、被安全策略拦截等，`message` 携带具体原因）；`401` 未登录或凭证失效；`403` 权限不足；`500` 服务器内部错误。

**鉴权约定**

- 采用 JWT 访问凭证，请求头携带 `Authorization: Bearer <token>`。
- 服务端将签发的 token 与密码版本号（passwordV）缓存于 Redis，校验时比对，改密后旧 token 立即失效。
- 访问凭证默认有效期 2 小时，刷新凭证默认 15 天，均可配置。
- 写操作接口通过权限码（`@Perms`）控制，账号 `admin` 为超级管理员放行全部；其余账号按所属角色的权限标识集合逐项比对。
- 智能体发起的敏感操作（命令/工具/记忆写入等）在鉴权之上，额外经安全策略模块统一裁决。

## 三、模块详细设计

> 各功能模块对应 `3.x`，包含模块职责、核心类/服务设计、核心业务流程与异常处理。类图与时序图先以文字流程呈现，后续可按需补充图形。

### 3.1 用户认证与登录 详细设计

**模块职责：** 核验用户身份、签发与校验访问凭证、加载账号可见的菜单与权限。（对应 SRS 3.5.1）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| OpenController | 接收登录等开放请求 | login / captcha |
| AuthService | 登录核验、凭证签发与失效、权限加载 | login / logout / validateToken / getPermissions |
| AuthGuard | 校验请求 JWT，与 Redis 缓存 token、密码版本比对 | canActivate |
| PermsGuard | 按权限码校验操作权限 | canActivate |

**核心业务流程 — 登录：**

1. 接收账号、密码（规划中含图形验证码）。
2. 校验账号是否存在、是否禁用、密码是否匹配；不通过统一提示"用户名或密码错误"。
3. 签发访问凭证与刷新凭证，写入 Redis（连同密码版本号）。
4. 返回凭证与账号基础信息，前端据此加载可见菜单与权限标识。

**核心业务流程 — 请求鉴权：**

1. AuthGuard 提取 `Authorization` 头中的 token。
2. 校验 token 有效性，并与 Redis 中缓存的 token、密码版本比对；不一致抛未授权异常。
3. 通过后，PermsGuard 按接口标注的权限码比对账号权限（admin 直接放行）。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 账号或密码错误 | 拒绝登录 | code 400 + "用户名或密码错误" |
| 账号被禁用 | 拒绝登录 | code 400 + "用户已被禁用" |
| 凭证缺失或失效 | 拦截请求 | code 401 + "未授权访问，请重新登录" |
| 权限码不匹配 | 拒绝操作 | code 403 + "无权限访问~" |

### 3.2 组织管理 详细设计

**模块职责：** 维护部门、岗位、用户三类组织基础数据。（对应 SRS 3.5.2）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| DepartmentController / DepartmentService | 部门层级树维护 | list / add / update / delete |
| PositionController / PositionService | 岗位维护 | list / add / update / delete |
| UserController / UserService | 用户维护、状态切换、批量删除 | page / add / update / updateStatus / delete / batchDelete |

**核心业务流程 — 新增用户：**

1. 校验必填项（姓名、账号、密码、所属部门），邮箱填写时校验格式。
2. 密码加密存储，写入用户记录并建立用户-角色、用户-部门关联。
3. 返回结果。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 删除有关联用户的部门/岗位 | 拒绝删除 | code 400 + "关联数据不存在或被引用" |
| 删除超级管理员账号 | 拒绝删除 | code 400 + 提示 |
| 用户状态切换失败 | 前端回滚开关 | code 400 + "状态更新失败" |

### 3.3 权限管理 详细设计

**模块职责：** 维护角色与菜单，及角色-菜单权限关联。（对应 SRS 3.5.3）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| RoleController / RoleService | 角色维护、分配权限 | list / add / update / delete / setMenus |
| MenuController / MenuService | 菜单层级树维护、显示切换 | list / add / update / updateStatus / delete |

**核心业务流程 — 角色分配权限：**

1. 接收角色 ID 与勾选的菜单/功能点 ID 列表。
2. 清空该角色原有的角色-菜单关联（全量覆盖）。
3. 按当前勾选结果重新写入关联记录。
4. 返回结果；该角色下用户的可见菜单与权限随之变化。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 角色标识重复 | 拒绝保存 | code 400 + "数据已存在（唯一约束冲突）" |
| 删除/更新目标不存在 | 中止操作 | code 400 + "记录不存在" |

### 3.4 模型配置 详细设计

**模块职责：** 维护模型服务商与模型，密钥加密存储与按需查看，接入地址安全校验与连接测试。（对应 SRS 3.5.4）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| ModelProviderConfigController / ModelProviderConfigService | 服务商配置维护、密钥加解密、连接测试 | list / add / update / revealKey / testConnection |
| ModelConfigController / ModelConfigService | 模型维护 | list / add / update / delete |
| CryptoUtil | API Key 对称加解密（AES-256-GCM） | encrypt / decrypt |
| UrlGuardUtil | 接入地址 SSRF 校验（协议/内网/域名解析） | assertSafeUrl |

**核心业务流程 — 连接测试：**

1. 取该连接已存密钥（解密）与接入地址。
2. UrlGuardUtil 校验地址：仅 http/https；禁本机、私有网段、链路本地、保留地址；域名先解析并逐一校验 IP；不跟随跳转。
3. 向地址发起探测请求（8 秒超时）。
4. 返回状态码 < 500 判定可达，5xx 或请求失败判定不可达。

**核心业务流程 — 查看密钥：**

1. 校验查看密钥的独立权限。
2. 解密返回明文，记录审计信息（谁/何时/查看了哪个配置），不记录明文，结果不缓存。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 接入地址指向内网 | 拒绝测试 | code 400 + "禁止访问内网地址" |
| 未配置密钥点击测试 | 前端拦截 | "该模型尚未配置 API Key，请先保存后再测试" |
| 同服务商下模型标识重复 | 拒绝保存 | code 400 + "该供应商下已存在相同模型 ID" |
| 服务器未配置加密口令 | 功能报错终止 | code 500 + 提示（不降级明文） |

### 3.5 AI 对话 详细设计

**模块职责：** 承载智能体对话运行、工具调用编排、运行时上下文与记忆注入、会话历史管理。（对应 SRS 3.5.5）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| AgUiController | 接收对话运行请求，流式输出 | run |
| AgUiService | 对话运行编排、上下文与记忆注入、工具调用循环 | runAgent / injectContext |
| ConversationController / ConversationService | 会话历史查询、保存、删除 | list / detail / save / remove |
| AgentMemoryService | 提供注入用的记忆内容、接收记忆建议 | getInjectableMemory / submitPending |

**核心业务流程 — 一轮对话运行：**

1. 校验消息非空、存在可用模型；取选定模型的服务商连接与密钥。
2. 注入当前页面结构化上下文、全局系统提示词、该用户启用且可读的记忆（超长截断）。
3. 调用模型流式生成；如返回工具调用，则做前端权限校验，高风险操作弹确认卡片。
4. 工具执行结果回传模型继续生成，循环最多 5 次，超过强制收尾一次。
5. 完成后保存会话正文消息（中间产物不落盘）。

**状态机（对话运行状态）：**

| 当前状态 | 触发条件 | 流转至 |
|---------|---------|-------|
| 空闲 | 发送消息 | 连接中 |
| 连接中 | 建连成功 | 运行中 |
| 运行中 | 接收文本 | 接收中 |
| 运行中 | 需工具执行 | 等待工具 |
| 运行中 | 需用户确认 | 已中断（待确认） |
| 接收中/等待工具 | 完成 | 已完成 |
| 任意运行态 | 用户停止 | 已停止 |
| 任意运行态 | 异常 | 运行失败 |

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 无可用模型 | 不发起请求 | "暂无可用模型，请先在「模型配置」中启用模型" |
| 调用未注册操作 | 拒绝执行 | "未注册的操作：{操作名}" |
| 调用无权限操作 | 拒绝执行 | "无权限执行该操作" |
| 模型鉴权失败 | 显示错误卡片 | "模型鉴权失败，请检查 API Key" |

### 3.6 对话设置 详细设计

**模块职责：** 按用户维护对话参数、系统提示词、界面偏好与快捷提示词模板。（对应 SRS 3.5.6）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| ChatSettingController / ChatSettingService | 用户对话设置读写（一人一行 upsert） | get / save |
| AgentConfigService | 全局配置（限制值、默认界面偏好），启动时幂等 seed 并缓存 | getConfig / getLimits / reload |

**核心业务流程 — 保存对话设置：**

1. 接收对话参数、界面偏好、模板集合（按用户 ID）。
2. 校验系统提示词长度（≤4000）、模板数量（≤50）与单条长度限制。
3. 采样温度、最大输出长度传入 null 表示清除恢复模型默认，字段缺省表示不改动。
4. 按用户 ID upsert 保存。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 系统提示词超长 | 拒绝保存 | code 400 + 提示 |
| 模板超量或单条超长 | 拒绝保存 | code 400 + 提示 |
| 最大输出长度超上限 | 按上限夹取 | 正常保存 |

### 3.7 记忆中心 详细设计

**模块职责：** 管理长期记忆文件及版本、待确认记忆、模型建议、命中率，并对五条写入路径统一做安全裁决。（对应 SRS 3.5.7）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| AgentMemoryController | 接收记忆管理请求 | list / detail / create / save / permission / toggle / delete / rollback / confirmPending / applySuggestion / check / recordRead |
| AgentMemoryService | 记忆文件增删改查、权限、启停、注入内容提供 | list / create / save / setPermission / toggle / remove / getInjectableMemory |
| MemoryVersionService | 版本快照与回滚 | bumpVersion / listVersions / rollback |
| MemorySecurityService | 五条写入路径统一安全裁决（fail-closed） | enforceWrite / enforceCreate |
| MemorySuggestionService | 待确认记忆与模型建议处理 | confirmPending / ignorePending / applySuggestion / ignoreSuggestion |
| MemoryReadlogService | 记忆读取留痕与命中率统计 | recordRead / hitRate |

**核心业务流程 — 编辑保存（含安全裁决）：**

1. 接收记忆文件 ID 与新内容。
2. MemorySecurityService.enforceWrite 委托安全策略统一裁决（综合风险等级、敏感词、需确认文件、审批规则）。
3. 裁决为拒绝则中止；需确认而未携带确认标识则拦截；需审批则生成审批工单待审；放行或已确认则继续。
4. 写入新内容，MemoryVersionService 记录版本快照（版本号单调递增）。

**状态机（记忆写入裁决）：**

| 裁决结果 | 系统动作 |
|---------|---------|
| 放行 | 直接写入并留版本 |
| 需确认 | 携带确认标识后写入，否则拦截 |
| 需审批 | 生成审批工单，通过后写入 |
| 拒绝 | 拦截并提示，不写入 |

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 需确认记忆未携带确认 | 拦截写入 | code 400 + 需确认提示 |
| 命中敏感内容 | 拦截写入 | code 400 + 命中原因 |
| 删除内置记忆文件 | 拒绝 | code 400 + 提示 |

### 3.8 安全策略 详细设计

**模块职责：** 统一治理智能体行为，提供风险等级、审批规则、黑白名单、敏感词、审计策略配置与统一安全裁决、审批工单流转、脱敏审计。（对应 SRS 3.5.8）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| SecurityPolicyController / SecurityPolicyService | 六类策略配置的读写 | risk / approvalRule / sandbox / list / sensitive / auditConfig 的增删改查 |
| SecurityCheckService | 统一安全裁决（唯一权威校验入口） | check / dryRun |
| ApprovalRequestService | 审批工单发起、审批、驳回、超时扫描 | create / decide / timeoutScan |
| MaskPayloadUtil | 审计与工单载荷脱敏 | maskPayload |

**核心业务流程 — 统一安全裁决（check）：**

1. 接收操作类型（cli/api/database/file/page/memory/skill）、风险等级、操作载荷。
2. 依次判定：CLI 危险命令直接拦截；命中黑名单拦截；命中敏感词按规则处理；数据库危险 SQL 升级风险等级。
3. 结合风险等级默认行为得出裁决：放行/需确认/需审批/拒绝。
4. 需审批则生成审批工单；高风险或被拦截操作写入脱敏审计日志。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 命中黑名单 | 拦截 | "命中黑名单：{详情}" |
| 命中 CLI 危险命令 | 直接拦截 | "命中 CLI 危险命令：{命中词}" |
| 命中敏感词 | 按规则拦截/脱敏 | "命中敏感词规则「{类型}」，阻止执行" |
| 风险等级被停用 | 拒绝 | "风险等级 {等级} 已停用" |

### 3.9 工具权限 详细设计

**模块职责：** 登记智能体可调用工具、分级与需确认配置，调用时委托安全裁决并留痕。（对应 SRS 3.5.9）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| AgentToolController / AgentToolService | 工具增删改查、启停、复制、调用裁决、调用日志 | list / create / update / toggle / copy / check / recordCallLog / callLogs |
| SecurityCheckService | 工具调用的最终执行许可裁决（委托） | check |

**核心业务流程 — 工具调用裁决（check）：**

1. 接收工具标识与调用动作。
2. 工具停用或未注册直接判不允许（未注册按最高风险处理）。
3. 数据库危险 SQL 升级风险等级；CLI 危险指令直接拦截。
4. 映射工具类型到操作类型，委托 SecurityCheckService 统一裁决得出最终许可。
5. 需审批生成工单；高风险/被拦截写审计。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 工具标识重复 | 拒绝创建 | code 400 + "工具键已存在" |
| 工具标识格式不符 | 校验拒绝 | code 400 + kebab-case 提示 |
| 工具停用时被调用 | 判不允许执行 | "工具已禁用，禁止执行" |

### 3.10 Skills 管理 详细设计

**模块职责：** 登记、配置、治理技能，版本管理与回滚、执行前安全校验、运行留痕、导入导出。（对应 SRS 3.5.10）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| AgentSkillController / AgentSkillService | 技能增删改查、启停、导入导出 | list / create / update / toggle / remove / import / export |
| SkillVersionService | 版本快照与回滚 | snapshot / listVersions / rollback |
| SkillPortService | 技能导入导出的序列化与冲突处理 | exportSkills / importSkills |
| SkillTestService | 能力试运行（执行前经安全校验） | test |

**核心业务流程 — 更新技能（版本递增）：**

1. 校验技能键格式与能力合法性（能力须在能力目录中登记）。
2. 内置技能与技能键不可修改。
3. 写入更新，版本号自动递增（vX.Y.Z → vX.Y.(Z+1)）。
4. 生成一条版本快照（变更类型=更新，摘要列出变更字段）。

**核心业务流程 — 版本回滚：**

1. 取目标版本快照，覆盖当前技能可变字段。
2. 版本号继续递增（不倒退），生成回滚快照记录，形成可追溯链。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 技能键重复 | 拒绝创建 | code 400 + "技能键已存在" |
| 未选能力 | 拒绝保存 | code 400 + "至少选择一个能力" |
| 删除内置技能 | 拒绝 | code 400 + "内置技能不可删除，可将其停用" |
| 试运行命中安全拦截 | 不发起调用 | 返回拦截原因 |

### 3.11 运行日志 详细设计

**模块职责：** 多源聚合查询对话/技能/工具/记忆/错误/系统六类运行日志，只读，支持详情、标记已处理、导出。（对应 SRS 3.5.11）

**核心类/服务设计：**

| 类/服务名 | 职责 | 关键方法 |
|----------|------|---------|
| AgentRunLogController | 接收查询、详情、标记、导出请求 | list / detail / markHandled / export |
| RunLogQueryService | 多源归一化聚合、统一排序分页、分类计数 | query / detail / countByCategory |
| RunLogService | 统一表日志写入（对话/系统/错误） | record |

**核心业务流程 — 多源聚合查询：**

1. 接收筛选条件（类型/日期/智能体/状态/关键词）。
2. 指定类型时只查对应源，否则查四源（统一表 + 技能/工具/记忆三明细表）。
3. 各源数据库层模糊匹配（单源上限 2000 条），归一化为统一结构。
4. 合并后按开始时间统一倒序，统一分页；分类计数忽略类型条件但保留其他条件。

**异常处理：**

| 异常场景 | 处理策略 | 错误反馈 |
|---------|---------|---------|
| 标记非统一表记录 | 拒绝 | code 400 + "仅支持标记本表(run-)日志，或记录不存在" |
| 日志写入失败 | 不中断触发方流程 | 内部告警 |

> 说明：运行日志与安全策略模块的审计日志（`sys_agent_audit_log`）是相互独立的两套系统，数据来源不同，不可混用。

#### 4.3.39 sys_agent_memory_read_log（记忆读取日志）

## 五、API 详细设计

### 5.1 通用规范

**统一响应格式**（同 2.3，此处再明确）：

```typescript
interface ApiResponse<T> {
  code: number     // 200 成功；400 业务错误；401 未登录；403 无权限；500 服务器错误
  message: string  // 提示信息
  data: T          // 业务数据
}

// 分页 data 内层
interface PageData<T> {
  list: T[]
  pagination: { page: number; pageSize: number; total: number }
}
```

**接口风格约定**（以真实实现为准）：

- 基础系统模块（组织/权限/字典/任务等）：RESTful 风格，前缀 `/admin`，由 CRUD 基类提供 list/detail/add/update/update-status/delete/batch-delete。
- 登录与开放接口：前缀 `/admin/open`。
- 智能体各模块（记忆/安全/工具/技能/运行日志/对话设置）：采用 `POST /admin/<模块>/<动作>` 的动作式接口（RPC 风格），查询与写入均为 POST，动作名语义化（list/save/toggle/delete/rollback/confirm 等）。
- 对话运行：`POST /api/ag-ui`（流式输出）。

**通用错误码：**

| code | message | 场景 |
|------|---------|------|
| 401 | 未登录或Token失效 | 鉴权失败 |
| 403 | 权限不足 | 越权 |
| 400 | （具体业务提示） | 参数校验失败、唯一约束冲突、被安全策略拦截等 |
| 500 | 服务器内部错误 | 系统异常 |

### 5.3 关键接口详细设计

> 此处对跨模块、含事务或安全裁决的关键接口做详细设计；基础 CRUD 接口遵循基类统一约定，不逐一展开。

#### 5.3.1 登录 — POST /admin/open/login

- **用途：** 账号密码核验，签发访问凭证。
- **鉴权：** 开放接口。

**请求参数：**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|-----|------|------|------|------|
| username | body | string | 是 | 登录账号 |
| password | body | string | 是 | 密码 |
| captchaId / captcha | body | string | 否 | 图形验证码（规划中） |

**成功响应：** `data` 含访问凭证、刷新凭证与账号基础信息。

**错误码：** 账号/密码错误 → 400「用户名或密码错误」；账号禁用 → 400「用户已被禁用」。

#### 5.3.2 获取权限与菜单 — GET /admin/open/permmenu

- **用途：** 返回当前账号可见菜单树与权限标识集合，供前端构建路由与按钮权限。
- **鉴权：** 需登录。
- **成功响应：** `data` 含 menus（菜单树）、perms（权限标识数组）。

#### 5.3.3 发起对话运行 — POST /api/ag-ui

- **用途：** 发起一轮智能体对话，流式返回文本、推理、工具调用事件。
- **鉴权：** 需登录。

**请求参数：**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|-----|------|------|------|------|
| messages | body | array | 是 | 对话消息列表 |
| providerConfigId | body | int | 是 | 服务商配置 ID |
| modelId | body | int | 是 | 模型 ID |
| context | body | object | 否 | 当前页面结构化上下文 |
| sessionId | body | string | 否 | 会话标识 |

**处理要点：** 校验模型可用 → 注入页面上下文/系统提示词/可读记忆 → 流式生成 → 工具调用循环（≤5 次，高风险需确认）→ 保存会话正文。

**错误码：** 无可用模型 / 模型鉴权失败 / 模型配置不存在或未设密钥 → 对应错误卡片提示。

#### 5.3.4 记忆编辑保存 — POST /admin/agent-memory/save

- **用途：** 保存记忆文件内容，写入前经安全策略统一裁决，成功后留版本。
- **鉴权：** 需登录 + 权限码。

**请求参数：**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|-----|------|------|------|------|
| id | body | int | 是 | 记忆文件 ID |
| content | body | string | 是 | 记忆正文 |
| confirmed | body | boolean | 否 | 是否携带人工确认标识 |

**处理要点（事务）：** 委托 MemorySecurityService 裁决 → 拒绝/需确认未确认则中止；需审批则建工单待审 → 放行则写入内容并生成版本快照（版本号递增）。写入与版本快照在同一事务内。

**错误码：** 需确认未确认 → 400 需确认提示；命中敏感内容 → 400 命中原因；内置记忆改标识 → 400 拒绝。

#### 5.3.5 统一安全裁决 — POST /admin/security-policy/check（内部基础能力）

- **用途：** 对智能体操作给出统一裁决，是工具权限、记忆写入的权威校验入口。
- **鉴权：** 需登录（由内部服务调用）。

**请求参数：**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|-----|------|------|------|------|
| actionType | body | string | 是 | cli/api/database/file/page/memory/skill |
| riskLevel | body | string | 是 | L1-L4 |
| payload | body | object | 否 | 操作载荷 |

**成功响应：** `data` 含裁决结果（allow/require_confirm/require_approval/deny）、命中策略、拦截原因（如有）。

**处理要点：** CLI 危险命令直接拦截 → 黑名单拦截 → 敏感词处理 → 数据库危险 SQL 升级 → 结合风险默认行为定论；需审批建工单，高风险/拦截写脱敏审计。

#### 5.3.6 工具调用裁决 — POST /admin/agent-tool/check（内部基础能力）

- **用途：** 工具执行前的许可裁决。
- **处理要点：** 工具停用/未注册直接判不允许（未注册按最高风险）→ 映射工具类型到 actionType → 委托 5.3.5 统一裁决。

#### 5.3.7 技能版本回滚 — POST /admin/agent-skill/version/rollback

- **用途：** 将技能回滚到指定历史版本。
- **处理要点（事务）：** 取目标版本快照覆盖当前技能可变字段 → 版本号继续递增（不倒退）→ 生成一条 rollback 类型版本快照。当前版本不可回滚到自身。

#### 5.3.8 运行日志聚合查询 — POST /admin/agent-run-log/list

- **用途：** 多源聚合查询运行日志。
- **处理要点：** 指定类型只查对应源，否则查四源；各源库层过滤（单源上限 2000）后归一化，合并按开始时间倒序、统一分页；技能/工具/记忆三源仅成功/失败状态，且无智能体维度。

## 六、安全设计

| 安全维度 | 设计 |
|---------|------|
| 鉴权 | JWT 访问凭证 + 刷新凭证，token 与密码版本号缓存于 Redis 比对；改密后旧 token 立即失效；同账号重登覆盖旧凭证 |
| 权限控制 | 权限码（`@Perms`）+ 角色-菜单关联；admin 超管放行全部；数据权限经角色-部门关联（relevance）约束可见范围 |
| 智能体行为治理 | 命令/工具/记忆写入等操作在鉴权之上，额外经安全策略统一裁决（风险等级 + 黑白名单 + 敏感词 + 审批规则）；CLI 危险命令直接拦截，高风险需确认或审批 |
| 密钥保护 | 模型 API Key 以 AES-256-GCM 加密存储，查询接口只返回"是否已配置"，查看明文需独立权限并记审计（不记明文、不缓存）；服务器未配加密口令时功能报错终止，不降级明文 |
| SSRF 防护 | 模型接入地址连接测试前经 URL 安全校验：仅 http/https；禁本机/私有网段/链路本地/保留地址；域名解析后逐一校验 IP；不跟随跳转 |
| 输入校验 | 各接口 DTO 参数校验（必填/长度/格式）；唯一约束冲突统一转友好提示 |
| 敏感数据 | 密码 bcrypt 哈希不返回；手机号列表脱敏展示；审计日志与审批工单载荷脱敏（MaskPayloadUtil），不落明文 |
| 记忆写入安全 | 五条写入路径 fail-closed，统一经安全裁决；需确认/需审批的记忆写入被拦截或转工单 |
| 多租户隔离 | 多数业务表含 tenant_id，按租户隔离数据 |
