# 卡通角色插画一键抠透明底 — 通用需求提示词

> 用法：需要把一张「卡通角色 + 纯色/渐变背景（蓝天、草地、花朵等）」的插画抠成透明底 PNG、
> 用作欢迎页/空状态插画时，直接把下方【提示词正文】整段复制粘贴给 AI 编码助手即可。
> 提示词不绑定具体项目，AI 会自动探测环境（是否可用 Python / Pillow / rembg）并适配。
>
> 适用场景：设计稿导出的角色插画自带背景，但页面需要透明底叠加在自定义光晕/渐变衬底上。
> 典型产物：一张干净透明、体积可控（几十 KB）的 RGBA PNG，接入前端 `<img>` 即可。

## 提示词正文（复制以下全部内容）

请写一个 Python 脚本，把我提供的「卡通角色插画」从其纯色/渐变背景（如蓝天、草地、花朵）中抠出，输出**透明底 RGBA PNG**，并接入前端欢迎页作为插画。请先探测环境是否可用 Python 与 Pillow；再按下面的分层方案实现，不要只调阈值硬试。

一、抠图算法（两级方案，rembg 首选、洪水填充回退）
1. **首选 rembg**：若环境已安装 `rembg`，用 `isnet-general-use` 模型（对卡通/线稿边缘更干净）去背景。模型需联网下载（约 179MB），网络差时会很慢——首次运行给足超时；下不动就自动回退到方案 2。
2. **回退：边缘区域生长洪水填充（不依赖网络/模型）**，这是本方案的核心，要点如下：
   - **从图像四条边界的像素做种子**（渐变天空/草地一定贴边），而不是只从固定几个角点，避免渐变背景漏填。
   - **区域生长时与「来源像素」比色，而非与固定种子比**：背景是平滑渐变，相邻像素色差很小（实测 ≤33），能连续蔓延；单步容差取 `step_tol=36`（介于"背景相邻色差"与"黑描边跳变"之间）。
   - **关键：把角色的黑色描边当作"不可穿越的墙"**。判定 `is_wall`：某像素三通道都 `< wall_max(=90)` 视为黑边。填充遇到墙时"标记已访问但不清除、也不越过"。卡通角色的黑描边是闭合回路，物理上挡住背景从外部漏进角色内部（蓝身、白脸和天空一样平滑，只调阈值必然级联吃穿角色）。

二、抠图后处理（三步，缺一不可）
1. **去背景孤岛（最大连通域保留）**：填充后，云朵/花朵等未与图像边缘相连的背景块仍是不透明的。用 4 连通标记所有不透明像素，只保留**面积最大的连通域**（角色主体），其余置透明。注意别误删胡须/铃铛等细节——它们通过黑描边与主体相连，不会被剔除。
2. **去边缘杂色（defringe）**：黑描边与背景之间的抗锯齿混合像素（如黑+绿草地混出的偏绿像素）绿通道常 >90 不算墙、又与背景差异 >36 没被填充，会残留成悬空色斑。做法：迭代若干遍（如 2 遍），把**"贴着透明区、且本身不是黑描边"的不透明像素**剥掉——既清杂色又保留黑色轮廓作边界。
3. **自动裁剪（autocrop）**：按不透明区域 `getbbox()` 裁掉多余透明边距，留少量 padding（如 12px）；`getbbox()` 为空要提前返回，裁剪坐标做边界钳制防越界。

三、体积与接入优化
1. **降采样**：设计稿常是超大图（如 779×1002）。按前端实际展示宽度的 2x 密度降采样即可（如展示 260px → 导出 520px 宽），用 LANCZOS 重采样。
2. **压缩**：卡通图是大片纯色，做 256 色调色板量化（`quantize(colors=256, method=FASTOCTREE, dither=NONE)`）+ `optimize=True`，近无损且体积可降 ~90%（实测 434KB→35KB）。量化后要复核透明度未被破坏。
3. **透明像素 RGB 清零**：`alpha==0` 的像素把 RGB 也置 0，防止缩放/合成时边缘色溢出。
4. **接入前端**：把成品替换到欢迎页的插画资源；`<img>` 的 `width`/`height` 属性要按**新图真实宽高比**设置（避免加载时布局抖动 CLS 与占位变形）；透明插画适合叠在浅色光晕/径向渐变衬底上。

四、验证（必须做，不能只看预览）
1. 很多图片预览器**忽略 alpha 只显示 RGB**，会让人误以为背景还在。务必把成品**合成到品红等鲜艳纯色底**上导出再肉眼检查，才能看清真实透明区与是否有残留光晕。
2. 抽样检查四角 `alpha==0`、角色中心 `alpha==255`；统计透明占比是否合理；统计"偏绿等杂色的不透明像素数"应为 0。
3. 参数（`step_tol=36`、`wall_max=90`）要基于**实测色差**确定，不要拍脑袋；不同插画可微调。
4. 清理所有临时验证文件（合成图等），不留痕迹。

质量要求：写完调用代码审查（重点：洪水填充边界情况与性能、连通域是否误删细节、Vue/前端资源引用与宽高比、图片体积）；产物用品红底合成验证通过后再交付。

## 参考实现（本仓库已落地，可作为范例）

| 部分 | 位置 |
|------|------|
| 抠图脚本（rembg 首选 + 洪水填充回退） | `scripts/cutout_welcome.py` |
| 透明底成品（520×669，~35KB RGBA） | `frontend/src/assets/agent/agent-welcome.png` |
| 欢迎页组件（插画 + 浅蓝光晕衬底） | `frontend/src/components/core/layouts/art-agent-chat/widget/AgentWelcome.vue` |

### 脚本函数结构（`cutout_welcome.py`）
- `try_rembg(src, dst)` — 首选，`isnet-general-use` 模型；环境无 rembg 或下载超时则抛异常触发回退。
- `try_floodfill(src, dst)` — 回退核心：四边界种子 + 与来源像素比色的区域生长（`step_tol=36`）+ 黑描边当墙（`wall_max=90`）阻挡。
- `keep_largest_blob(im)` — 4 连通标记，只留最大不透明连通域，去云朵/花朵孤岛。
- `defringe(im, passes=2, wall_max=90)` — 剥掉贴透明区的非黑描边像素，清抗锯齿绿边。
- `autocrop(im, pad=12)` — 按不透明 bbox 裁剪留边。
- `main()` — `try_rembg` → 失败 `except` 回退 `try_floodfill` + `keep_largest_blob` + `defringe`，最后统一 `autocrop` 保存。

### 落地关键决策（复用时参考）
- **只调阈值治不了卡通抠图**：角色内部（蓝身/白脸）和天空一样平滑，任一处描边缺口都会让填充级联吃穿整块区域。真正的解法是**把黑色闭合描边当物理墙**。
- 参数是**实测**出来的：背景相邻像素色差 ≤33、黑描边跳变 >400 → 单步容差取中间的 36；黑边阈值 90。
- **rembg 是理想解但受网络制约**：模型 179MB，弱网下 2 小时也下不完；脚本保留 rembg 首选逻辑，网络好时重跑自动启用，当前以 floodfill 成品交付。
- **预览器会骗人**：多数预览忽略 alpha 只显 RGB，看起来"背景还在"其实已透明；一律用**品红底合成**验证真实透明度。
- **体积**：降采样到展示 2x + 256 色量化，434KB→35KB，且首屏 `loading="eager" fetchpriority="high"` 的关键图不拖慢加载。
- **接入**：`<img>` 的 `width`/`height` 必须匹配新图真实比例（本例 779×1002 ≈ 520:669，属性写 `height="669"`），CSS 层 `width: clamp(...); height:auto` 只改显示尺寸不影响固有比例，避免 CLS。
