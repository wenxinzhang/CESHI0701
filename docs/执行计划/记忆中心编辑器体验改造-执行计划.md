# 记忆中心编辑器体验改造 · 执行计划

> **版本**：v1.1　|　**创建**：2026-07-14　|　**状态**：✅ 已实施完成（阶段 0–12 全部完成，待人工验收）
> **用途**：本文件是「智能体设置 → 记忆中心」布局与 Markdown 编辑/预览/分屏体验改造的唯一执行依据与进度台账。
> 每完成一个任务就勾选复选框并更新「§10 进度追踪总表」。中断后按「§12 中断恢复指引」定位继续。

## 1. 背景与结论（务必先读）

**任务性质**：**纯前端**布局与交互改造。后端 `agent-memory` 接口（list/detail/save/version/rollback/pending/suggestion 等）已全部就绪，**本次不动后端、不动数据库、不改任何接口契约、不影响其他设置页签**。

**核心痛点**（用户反馈）：记忆文件内容查看/编辑区域过小（当前仅一个小 `ElInput textarea`），阅读和编辑 Markdown 极不方便；弹窗与页面存在大量无效空白。

**现状盘点（复用优先）**：

| 能力 | 现状 | 结论 |
|---|---|---|
| UI 组件库 | Element Plus 2.11 + `@element-plus/icons-vue` + `iconfont-sys` 字体图标 | 复用 |
| 状态管理 | Pinia 3 + `pinia-plugin-persistedstate` | 复用 `agentMemory` store，扩展编辑态字段 |
| Markdown 渲染 | **已有** `utils/markdown/renderMarkdown.ts`（零依赖、先转义再白名单、防 XSS）+ `MarkdownRenderer.vue` | 复用；**补充引用块 `>`、分割线 `---`** 两条规则 |
| HTML 安全过滤 | **已有**：`renderMarkdown` 全文先 `escapeHtml` 再白名单拼装，链接协议白名单，外链已加 `target=_blank rel=noopener` | 复用，**无需引入 DOMPurify/markdown-it** |
| 代码高亮 | `highlight.js` 11 + `v-highlight` 指令（行号/复制按钮，暗色主题） | 预览区代码块复用；编辑器另走浅色主题 |
| Markdown 编辑器 | **无任何编辑器**（无 Monaco/CodeMirror/Vditor/md-editor-v3） | **新增 CodeMirror 6**（用户已确认） |
| 工具库 | `@vueuse/core` 13.9（`useLocalStorage`/`useEventListener`/`useDebounceFn`/`useDraggable` 等） | 复用于草稿/防抖/beforeunload/拖动 |
| 用户 ID | `useUserStore().getUserInfo.id`（`Api.Auth.UserInfo.id: number`） | 草稿 key 用 |

**关键涉及文件**：
- 弹窗容器：`frontend/src/components/core/layouts/art-agent-chat/widget/ModelConfigDialog.vue`（尺寸 80vw/82vh → 92vw/88vh）
- 记忆中心根：`.../widget/MemoryCenter.vue`（编排 + 未保存保护 + 草稿 + beforeunload）
- 详情面板：`.../memory-center/MemoryDetailPanel.vue`（**重构为编辑/预览/分屏三模式**）
- 统计卡片：`.../memory-center/MemoryStatsCards.vue`（压缩高度）
- 文件列表：`.../memory-center/MemoryFileList.vue`（宽度收窄 + 元信息）
- 版本弹窗：`.../memory-center/MemoryVersionModal.vue`（**增强：Markdown 预览 + diff 对比**）
- Store：`frontend/src/store/modules/agentMemory.ts`（扩展编辑态/草稿/dirty）
- 共享渲染器：`frontend/src/utils/markdown/renderMarkdown.ts`（补引用块/分割线）

## 2. 已确认关键决策（用户已拍板）

| # | 决策项 | 选定方案 |
|---|--------|---------|
| D1 | Markdown 编辑器引擎 | **CodeMirror 6**：仅作源码编辑器（行号/语法高亮/当前行/查找/撤销重做/光标位置/浅色主题），预览复用现有 `renderMarkdown`。不引入自带渲染器的一体化编辑器 |
| D2 | 预览与安全过滤 | **复用现有 `renderMarkdown`**（已防 XSS），补引用块 `>`、分割线 `---`；**不引入** markdown-it / DOMPurify |
| D3 | 历史版本 diff 对比 | **本次做**：历史版本弹窗内支持「查看快照 Markdown 渲染」+「与当前版本逐行 diff 对比」（自研轻量行级 LCS，无新依赖） |
| D4 | 分屏同步滚动 | **本次做**：分屏内提供「同步滚动」开关，按滚动比例联动两侧，加锁去双向死循环 |
| D5 | 视图模式与分屏比例持久化 | 存 `localStorage`（`useLocalStorage`），key 全局单值：`memory-view-mode`、`memory-split-ratio` |
| D6 | 本地草稿 | `localStorage`，key = `memory-draft:{userId}:{fileId}`（无独立 agentId 概念，用 userId 隔离用户） |
| D7 | 保存路径 | 复用现有 `store.saveContent` → `/admin/agent-memory/save`；个人文件安全预判已放行，`guardedWrite` 逻辑保留但对个人文件直接放行 |

## 3. 依赖与技术约定

**新增依赖**（CodeMirror 6，均为活跃维护的官方分包）：
```
codemirror                    # 元包（含 basic-setup：行号/当前行/历史/查找等）
@codemirror/lang-markdown     # Markdown 语法高亮
@codemirror/language-data     # 代码块内嵌语言高亮（可选，按需）
@codemirror/view @codemirror/state @codemirror/commands  # 通常随元包传递依赖，显式声明便于按需 import
```
> 说明：`codemirror` 元包已内置行号(`lineNumbers`)、当前行高亮(`highlightActiveLine`)、撤销/重做(`history`)、查找(`search`/`openSearchPanel`)、Tab 缩进(`indentWithTab`)。浅色主题用默认（不引入 `oneDark`）。安装命令用 `pnpm`（项目 `engines` 声明 pnpm）。

**约定**：
- 所有颜色/边框/圆角走 `--art-*` CSS 变量，禁止硬编码主题色（与现有组件一致）。
- 图标沿用 `iconfont-sys` 字体图标（与现有工具栏/按钮一致）。
- 组件 `defineOptions({ name })`、JSDoc 注释、`<!-- PART_SCRIPT -->`/`<!-- PART_STYLE -->` 分节风格与现有文件对齐。
- 类型全部走 `memory-constants.ts` 契约，不新造重复类型。

## 4. 目标布局（对齐设计图）

```
ElDialog（92vw / 88vh，min 1100px / max 1600px，居中，整体不滚动）
├─ 标题「智能体设置」                          [固定]
├─ ElTabs（模型配置 / Skills / 记忆中心 / 工具权限 / 运行日志 / 安全策略）  [固定]
└─ 记忆中心（memory-center，flex column，min-height:0）
   ├─ 统计卡片 ×5（高度 64–76px，紧凑）        [固定]
   └─ 工作区（mc-body，flex:1，min-height:0）
      ├─ 左栏（250–280px，flex column）
      │   ├─ 文件列表（独立滚动，含文件名/说明/启用/更新时间）
      │   └─ 待确认记忆（独立滚动）
      └─ 右栏（flex:1，min-width:0，视觉主体）
         ├─ 文件信息头（图标/名/说明/创建人·时间/更新人·时间/版本 + 未保存标签）
         ├─ 操作行（版本选择 / 历史版本 / 保存 / 取消 / 更多） + 模式切换 [编辑|预览|分屏]
```

关键 CSS：每层 flex 容器都设 `min-height:0`（防子元素撑破无法滚动）；弹窗 body `overflow:hidden`；仅左列表、右内容区各自 `overflow:auto`。杜绝双滚动条与横向滚动条。

## 7. 需求合理性调整说明（与原提示词的差异）

| 原需求 | 调整 | 理由 |
|---|---|---|
| §13 引入 HTML sanitize 组件 | **不引入**，复用现有 `renderMarkdown` 白名单 | 现有渲染器已"先转义后拼装"防 XSS，引入 DOMPurify 属重复且与"白名单不解析原生 HTML"策略冲突 |
| §12 预览引入渲染器 | 复用现有零依赖渲染器 | 满足标题/列表/表格/引用/代码块/任务列表，仅补引用块与分割线两条规则 |
| §10 编辑器多特性 | CodeMirror 6 一次满足行号/高亮/当前行/查找/撤销重做/光标 | 避免自研难以达标，且不引入多个编辑器 |
| §六 左栏 250–280 / §current 380 | 取 **264px** | 落在建议区间中值，兼顾文件名与更新时间展示 |
| §18 草稿 key 含 agentId | key 用 `{userId}:{fileId}` | 系统当前无独立 agentId 概念，userId 已足够隔离 |
| §20 版本对比 | 自研轻量行级 diff | 不为 diff 引入重依赖 |

> 若后续引入多智能体，草稿 key 预留可平滑加 agentId 段。
