## 1. 背景与结论（务必先读）

**关键事实**：记忆中心前端 UI **100% 已完成但全部是前端 mock**，后端**零实现**。本计划是「从零开发后端 + 数据库设计 + 前端接线真实接口 + 跨模块业务联动」。

**前端现状**（`frontend/src/components/core/layouts/art-agent-chat/widget/memory-center/` + `store/modules/agentMemory.ts`）：
- 6 个核心实体：记忆文件(MemoryFile)、版本记录(MemoryVersion)、模型建议(ModelSuggestion)、权限配置(MemoryPermission)、待确认记忆(PendingMemory)、统计概览(MemoryStats)
- 数据由 `memory-constants.ts` 的 `createMemoryFiles()` / `createPendingMemories()` / `createMemoryStats()` 提供 mock，`agentMemory.ts` store 维护交互态
- 5 个内置记忆文件：soul.md(身份)、user.md(画像)、memory.md(项目记忆)、skill-memory.md(技能经验)、tool-memory.md(工具经验)
- 交互操作：选中/编辑保存/权限更新/版本回滚/确认待确认/忽略待确认/应用模型建议/忽略建议（共 10 个 store action，全部内存态）
- `onCreate`（新建记忆文件）当前是假的 toast「即将上线」

**后端现状**：`server/`（NestJS 11 + Prisma 6）**无 agent-memory 模块、无任何记忆表**。但存在可直接复用的联动基础（见下）。

**重大联动线索（系统预留）**：安全策略模块 `SecurityCheckService` **已内置 `actionType='memory'` + `memoryFile` 上下文 + `CONFIRM_MEMORY_FILES=['soul.md','user.md']`**；审计表 `sys_agent_audit_log.actionType` 枚举含 `memory`。说明「记忆写入接入安全治理」是系统设计时就预埋的钩子，本计划负责把它接通。

**本计划范围**：
1. 数据库设计——新建 5 张表（记忆文件/版本/待确认/模型建议/读取日志）
2. 后端开发——新建 `agent-memory` 模块（controller/service/dto/enums），约 20 个接口
3. 前端接线——把 `agentMemory.ts` store 的 mock 全部替换为真实 API
4. 业务联动——记忆写入统一过 `SecurityCheckService`（按 riskLevel 放行/需确认/需审批 + 审计）
5. 设计补缺——命中率真实来源（读取日志表）、记忆文件 riskLevel 分级、新建/删除记忆文件

## 2. 已确认的关键决策（用户已拍板，不可擅自更改）

| # | 决策项 | 选定方案 | 影响 |
|---|--------|---------|------|
| D1 | 记忆写入安全联动 | **接入安全策略统一治理**：保存编辑/确认待确认/应用建议/自动写入前统一过 `SecurityCheckService.check(actionType='memory')`，按 riskLevel 放行/需确认/需审批，auditRequired 时写审计日志 | 阶段 5 核心；复用 memory 钩子 |
| D2 | 待确认记忆 / 模型建议来源 | **存储 + 人工/种子产生**：本期做完整存储与管理（增删/确认/应用/忽略），数据由种子初始化 + 预留生成接口。真实「从对话自动提炼」作为独立后续能力，不在本期 | 阶段 3/4 只做存储管理，不做 LLM 提炼 |
| D3 | 记忆命中率数据来源 | **新建记忆读取日志表** `sys_agent_memory_read_log`：对话时记忆被注入即记一条，命中率=被引用记忆数/总读取。真实可信 | 阶段 6 新增读取埋点接口 |
| D4 | 权限存储方式 | **扁平列存储**（canRead/canSuggest/canAutoWrite/needConfirm/auditLog 各一列，enabled 复用文件启用列）| 便于「高风险记忆数」等统计直接 SQL 查询 |
| D5 | 记忆文件风险分级 | 记忆文件加 `riskLevel` 列（L1-L4，与安全策略统一）。soul.md/user.md 默认 L3（身份/画像敏感），其余 L1/L2 | 驱动 stats「高风险」卡片 + 安全 check 风险判定 |
| D6 | 内置记忆文件保护 | 5 个内置文件 `builtin=true`，不可删除、不可改 memoryKey；可编辑内容/权限（同 agent-skill 内置技能模式）| 阶段 2 删除接口拦内置 |
| D7 | 版本快照时机 | 保存编辑/确认待确认/应用建议/回滚 均写版本快照（changeType 区分），回滚不物理删历史（形成可追溯链）| 阶段 3 复用 agent-skill version 模式 |

## 3. 全局技术约定（对齐现有代码，尤其 agent-skill 模块）

| 项 | 约定 |
|----|------|
| 后端根目录 | `server/` |
| 参照模板 | `server/src/modules/agent-skill/`（结构/版本回滚/安全联动最接近，直接照搬骨架） |
| ORM | Prisma 6，`schema.prisma` 单文件，`///` 注释同步为列 comment |
| 表命名 | 模型 PascalCase + `@@map("snake_case")`；列驼峰不加 `@map` |
| 主键 | `id Int @id @default(autoincrement())` |
| 公共字段 | `tenantId Int?` + `createTime DateTime @default(now())`（需更新时加 `updateTime DateTime @updatedAt`），无软删除 |
| Controller | `@Controller('admin/agent-memory')`，全 `@Post` + `@Perms('action')`，继承 `BaseController`，`this.ok(data)` |
| Service | 继承 `BaseService`，构造注入 `PrismaService`（`super(prisma, 'sysAgentMemoryFile')`）；单文件≤500 行，超了拆子 service |
| DTO | class-validator + `@ApiProperty`，枚举字段 `@IsIn` |
| 权限点 | 新增 action 经 PermsSyncService 自动登记为按钮；需确认锚点菜单 `agent-memory:list` 存在（阶段 0 校验） |
| 安全联动 | 复用 `SecurityCheckService.check()`，模块 import `SecurityPolicyModule`（同 agent-skill 阶段5 做法） |
| 前端 API 层 | `frontend/src/api/agentMemory.ts`（新建），`import request from '@/utils/http'` |
| 前端 store | 改造 `frontend/src/store/modules/agentMemory.ts`：mock 初始化 → 真实接口拉取，action 内改调 API |
| schema 变更 | `cd server && npm run prisma:push`（同步 `///` 注释到列 comment），补迁移文件 |
| 后端验证 | `cd server && npx tsc --noEmit` |
| 前端验证 | `cd frontend && npx vite build`（含 vue-tsc） |
| 单文件行数 | ≤500 行；单函数≤80 行（超即拆） |

## 4. 数据库设计（5 张新表，全部新建）

### 4.0 表总览

| # | 表名 | 用途 | 对应前端实体 | 所属阶段 |
|---|------|------|-------------|---------|
| T1 | `sys_agent_memory_file` | 记忆文件主表（内容/权限/风险/关联） | MemoryFile + MemoryPermission | 阶段 0 建表 / 阶段 1-2 逻辑 |
| T2 | `sys_agent_memory_version` | 版本历史快照（支撑历史展示 + 回滚） | MemoryVersion | 阶段 0 建表 / 阶段 3 逻辑 |
| T3 | `sys_agent_memory_pending` | 待确认记忆候选 | PendingMemory | 阶段 0 建表 / 阶段 4 逻辑 |
| T4 | `sys_agent_memory_suggestion` | 模型对记忆文件的写入建议 | ModelSuggestion | 阶段 0 建表 / 阶段 4 逻辑 |
| T5 | `sys_agent_memory_read_log` | 记忆读取/注入日志（支撑命中率统计） | 无（后端补缺口，D3） | 阶段 0 建表 / 阶段 6 逻辑 |

### 4.1 T1 `sys_agent_memory_file`（记忆文件主表）

| 列 | 类型 | 说明 |
|----|------|------|
| id | Int PK autoincrement | 主键 |
| memoryKey | String @unique @db.VarChar(64) | 记忆文件唯一键=文件名，如 `soul.md`（对应前端 MemoryFile.id/name）|
| name | String @db.VarChar(100) | 文件名（=memoryKey，冗余便于展示）|
| description | String? @db.VarChar(200) | 简短描述（列表用，对应 desc）|
| subtitle | String? @db.VarChar(200) | 详情副标题（对应 subtitle）|
| content | String @db.Text | Markdown 正文内容 |
| category | String @default("internal") @db.VarChar(32) | 记忆分类（internal 内部记忆；预留 project/custom）|
| riskLevel | String @default("L1") @db.VarChar(8) | 风险等级 L1-L4（D5，与安全策略统一；soul/user 默认 L3）|
| version | String @default("v1.0.0") @db.VarChar(20) | 当前版本号（写入时递增修订号）|
| enabled | Boolean @default(true) | 是否启用（=permission.enabled，关闭后不参与对话）|
| canRead | Boolean @default(true) | 权限：允许模型读取 |
| canSuggest | Boolean @default(true) | 权限：允许模型建议修改 |
| canAutoWrite | Boolean @default(false) | 权限：允许低风险自动写入 |
| needConfirm | Boolean @default(true) | 权限：高风险写入需人工确认 |
| auditLog | Boolean @default(true) | 权限：记录审计日志 |
| relatedIds | Json? | 关联记忆文件 memoryKey 数组，如 `["user.md","memory.md"]` |
| builtin | Boolean @default(false) | 是否内置（内置不可删除、不可改 memoryKey，D6）|
| sort | Int @default(0) | 排序值（升序）|
| creator | String? @db.VarChar(100) | 创建者（用户名快照）|
| updater | String? @db.VarChar(100) | 最后更新人快照 |
| tenantId | Int? | 租户 ID |
| createTime | DateTime @default(now()) | 创建时间 |
| updateTime | DateTime @updatedAt | 更新时间 |

**索引/关系**：`@@index([tenantId])`、`@@index([riskLevel])`、`@@index([enabled])`；反向关系 `versions`/`suggestions`/`readLogs`；`@@map("sys_agent_memory_file")`

### 4.2 T2 `sys_agent_memory_version`（版本历史快照）

| 列 | 类型 | 说明 |
|----|------|------|
| id | Int PK | 主键 |
| memoryId | Int | 所属记忆文件 ID |
| version | String @db.VarChar(20) | 该快照版本号（如 v1.3.0）|
| content | String @db.Text | 该版本内容快照（回滚时恢复此内容）|
| changeType | String @db.VarChar(20) | 变更类型：create/update/rollback/confirm/suggestion |
| note | String? @db.VarChar(300) | 变更说明（对应前端 note，如"优化行为边界""回滚自 v1.2.0"）|
| updater | String? @db.VarChar(100) | 更新人快照 |
| tenantId | Int? | 租户 ID |
| createTime | DateTime @default(now()) | 快照生成时刻（=前端 time）|

**索引/关系**：`@@index([memoryId])`、`@@index([createTime])`、`@@index([tenantId])`；`skill/file @relation(onDelete: Cascade)`；`@@map("sys_agent_memory_version")`
> 前端 `MemoryVersion.current` 由「version==文件当前 version」派生，不落列（避免多行 current 一致性问题）。

### 4.3 T3 `sys_agent_memory_pending`（待确认记忆）

| 列 | 类型 | 说明 |
|----|------|------|
| id | Int PK | 主键 |
| text | String @db.VarChar(1000) | 建议记忆内容 |
| targetKey | String @db.VarChar(64) | 建议写入的目标文件 memoryKey |
| source | String? @db.VarChar(300) | 来源说明（如"来自 3 次页面设计对话的偏好归纳"）|
| status | String @default("pending") @db.VarChar(20) | 状态：pending/confirmed/ignored（保留处理痕迹，不物理删）|
| userId | Int? | 归属用户 ID |
| tenantId | Int? | 租户 ID |

| handleTime | DateTime? | 处理时间（确认/忽略时写入）|

**索引**：`@@index([status])`、`@@index([tenantId])`；`@@map("sys_agent_memory_pending")`

### 4.4 T4 `sys_agent_memory_suggestion`（模型建议）

| 列 | 类型 | 说明 |
|----|------|------|
| id | Int PK | 主键 |
| memoryId | Int | 所属记忆文件 ID |
| text | String @db.VarChar(1000) | 建议内容 |
| status | String @default("pending") @db.VarChar(20) | 状态：pending/applied/ignored |
| source | String? @db.VarChar(300) | 来源说明（可空）|
| tenantId | Int? | 租户 ID |
| createTime | DateTime @default(now()) | 创建时间 |

**索引/关系**：`@@index([memoryId])`、`@@index([status])`；`file @relation(onDelete: Cascade)`；`@@map("sys_agent_memory_suggestion")`

### 4.5 T5 `sys_agent_memory_read_log`（记忆读取日志，D3 命中率来源）

| 列 | 类型 | 说明 |
|----|------|------|
| id | Int PK | 主键 |
| memoryId | Int? | 被读取的记忆文件 ID（可空，未命中时记 null）|
| memoryKey | String? @db.VarChar(64) | 被读取记忆 key 快照 |
| sessionId | String? @db.VarChar(64) | 对话会话标识（用于按对话去重统计）|
| hit | Boolean @default(true) | 本次读取是否命中（记忆被实际注入=true）|
| userId | Int? | 用户 ID |
| tenantId | Int? | 租户 ID |
| createTime | DateTime @default(now()) | 读取时刻 |

**索引/关系**：`@@index([memoryId])`、`@@index([createTime])`、`@@index([tenantId])`；`file @relation(onDelete: Cascade, references id)`（memoryId 可空需用可空关系）；`@@map("sys_agent_memory_read_log")`
> **命中率定义**：`hitRate = 近7日 hit=true 的读取次数 / 近7日总读取次数`（无读取时返回「-」而非 0，避免误导）。

### 6.2 联动清单（改动某处需连带检查）

| 改动点 | 连带影响 | 涉及阶段 |
|--------|---------|---------|
| 记忆文件 `riskLevel` | 阶段5 check 风险判定；stats「高风险记忆数」卡片；写入是否需确认 | 2/5 |
