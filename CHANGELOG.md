# 版本更新记录（CHANGELOG）

本文件记录项目每次提交/发布的更新内容，最新在最上方，倒序排列。

## 维护约定

- **每次提交（commit）后**在下方「更新记录」区追加一条，简要总结本次改动。
- 每条记录包含：日期、简短标题、提交哈希（短），以及按 `新增 / 优化 / 修复 / 其他` 分组的要点。
- 标题风格与提交信息保持一致（`feat/fix/perf/docs/chore(scope): 描述`）。
- 只写“用户/维护者关心的变化”，实现细节留在代码与提交信息里，不在此展开。
- 分组无内容的可省略；一次提交跨多个模块时按模块分点。

### 2026-07-13 · 记忆中心 Markdown 编辑器改造与编辑安全加固

`feat(memory-center)` · 提交 `162b54c`

**新增**
- 记忆中心接入 CodeMirror 编辑器与 Markdown 工具栏，支持「编辑 / 预览 / 分屏」三模式。
- 详情面板拆分为头部、编辑工作区、权限弹窗等子组件；新增行级 diff 工具支撑版本对比。

**优化**
- 进入记忆中心默认以只读「预览」态呈现，需要修改再切换到编辑/分屏。
- 头部操作区重排为「版本 / 辅助 / 主操作」三组分栏，次要按钮弱化；预览态隐藏保存/取消按钮（存在未保存修改时仍显示）。

**修复**
- 编辑中切换设置页签再切回，不再静默丢弃未保存修改（改为轻量刷新，不重置选中与编辑缓冲）。
- 新建文件前增加未保存修改二次确认；创建成功才重置编辑缓冲，失败则保留用户编辑。
- 新建记忆后按 key 显式选中新文件，消除“选中新文件但编辑缓冲仍停留在列表首项”的不一致。
- 后端：个人记忆文件写入预判（checkWrite）直接放行，与实际写入路径一致，避免高风险文件被误判为“需审批”而拦下本人可自主完成的写入。
